# V1.12AO CPO Role-Layer Patch Pilot V1

- This pass exists to answer one question:
  - can the current role layer improve without widening the pilot
- Added patch families:
  - `role_patch_microstructure_family`
  - `role_patch_behavior_family`
- Boundaries:
  - no widened geometry
  - no formal training promotion
  - no formal signal generation
- Success interpretation:
  - if `role_state` improves, the project should prefer patch-before-widen
  - if it does not, the project should not pretend the next step is generic training expansion
