# V1.12Q CPO Information Registry Schema

`V1.12Q` hardens the earlier broad CPO registry into a more traceable schema. The point is not to make the line cleaner by deleting noisy rows; the point is to make omissions easier to notice, explain, and later review.

The schema adds:

- explicit pre-rise watch coverage
- a stronger layer split for technical, role, branch, liquidity, and noise
- A-E buckets so core leaders do not collapse into spillover rows
- review-first feature slots for each layer
- bounded subagent collection routes for repetitive work

This phase remains foundation-only. It does not open training, feature promotion, execution, or signal logic.
