# 111 V15 Local Regime Segmentation Candidate

## Status
- candidate_only
- not_opened
- no_phase_switch_yet

## Why This Candidate Exists
- Global structure work in `V1.2` showed that many lanes still collapse into `opening-led` outcomes instead of repairing the `carry_row_diversity_gap`.
- `V1.3` built bounded catalyst and concept context infrastructure, but that phase stopped at infrastructure closure rather than context consumption or local-model work.
- A later phase may need to test whether some currently "uncontrolled" residuals are really regime-mixing problems rather than missing global features.

## Candidate Mission
- Test whether bounded local segmentation can reduce residual noise by separating materially different market environments before any broader model expansion.

## Intended Reading
- This is not "train many sector models."
- This is "define bounded local regimes, then test whether some local slices are cleaner than the current global view."

## Preferred Segmentation Axes
- catalyst persistence class
- concept confirmation depth
- opening-heavy vs carry-capable context
- breadth / leader-only market confirmation
- strength-state buckets rather than static board names

## Explicit Non-Goals
- no direct strategy integration
- no wide local-model grid search
- no sector-name-first model explosion
- no promotion to retained feature or mainline signal without separate evidence

## Preconditions Before Opening
- a later phase must first show that bounded context consumption is actually useful
- sample counts must be large enough to avoid trivial slice overfitting
- local slices must be defined by auditable rules, not by ad hoc case selection

## Why It Is Deferred
- The current repo is in waiting state after `V1.2` and `V1.3`.
- No owner-approved phase switch has opened local segmentation work yet.
- This candidate should stay behind a likely context-consumption phase rather than leapfrog infrastructure closure.

## Likely Position In The Roadmap
- plausible follow-on candidate after a future `V1.4 Context Consumption Pilot`
- stronger fit for a later phase such as `V1.5 Local Regime Segmentation And Bounded Local Models`

## Opening Conditions
- owner phase switch
- bounded charter
- explicit success and stop criteria
- replay-independent or tightly bounded pilot scope
