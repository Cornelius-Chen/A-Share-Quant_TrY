# A-Share Quant Postmortem Log

## 目的

本文件记录严重异常、错误结论、数据污染、实现缺陷和策略失效事件。

工程级项目不仅要记录成功，也要系统记录失败。

---

## 需要记录的事件

1. 回测引擎 bug 导致结果失真
2. 数据污染或字段时间语义错误
3. 成本、涨跌停、成交模型假设错误
4. 指标计算错误
5. 协议升级后历史结论被推翻
6. Shadow / live 与 backtest 偏差异常
7. 策略在关键阶段失效
