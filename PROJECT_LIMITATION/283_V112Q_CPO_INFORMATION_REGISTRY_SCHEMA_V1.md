# V1.12Q CPO Information Registry Schema V1

`V1.12Q` freezes one CPO registry schema with:

- `6` cycle stages from `pre_ignition_watch` through `divergence_and_decay`
- `9` information layers
- `5` object buckets
- review-first feature slots across the layers
- bounded subagent collection tasks for repetitive draft collection

The most important structural upgrade is the explicit `pre_ignition_watch` stage. This reflects the owner judgment that collection should start slightly before the visible rise because many useful CPO clues appear before the markup becomes obvious.

The schema also distinguishes:

- core leader and mid-core seeds
- adjacent catch-up names
- branch and supply-chain extensions
- weak-relevance or name-bonus spillover rows
- board, liquidity, and event-route rows

This is still a review and collection foundation. The schema is not a training authorization surface.
